# Workflow: Implement a SOAP Interface

In this workflow, you create an API using Studio and APIkit for SOAP. APIkit for SOAP supports consuming a remote WSDL. This workflow, however, uses a local WSDL that you download. The SOAP interface you create accesses data resources and integrates the interface with backend processes that function as an API.

1. [Meet prerequisites for using APIkit for SOAP](https://docs.mulesoft.com/apikit/3.x/apikit-soap-prerequisites-task)
2. [Create an APIkit for SOAP project](https://docs.mulesoft.com/apikit/3.x/apikit-soap-project-task)
3. [Generate a SOAP fault](https://docs.mulesoft.com/apikit/3.x/apikit-soap-fault-task)
4. [Prevent the SOAP fault](https://docs.mulesoft.com/apikit/3.x/apikit-prevent-fault-task)
5. [To get a SOAP header](https://docs.mulesoft.com/apikit/3.x/apikit-get-header-task)
6. [To set a SOAP header](https://docs.mulesoft.com/apikit/3.x/apikit-set-header-task)
7. [To display a typed fault](https://docs.mulesoft.com/apikit/3.x/apikit-display-fault-task)
8. [To Update a WSDL](https://docs.mulesoft.com/apikit/3.x/apikit-update-wsdl-task)
